# 梳理工具箱使用教程

这份文档用于录制后台使用教程。录屏时可以按章节顺序讲，每一段先说明用途，再实际操作一次。

## 1. 项目介绍

梳理工具箱后台用于集中管理工具箱客户端内容。后台可以配置工具箱标题、主题、按钮、下载地址、用户账号和专属客户端。

后台主要分为这些模块：

- 总览：配置工具箱基础信息、主题、启动密码、更新入口和 EXE 属性。
- 账号：修改当前登录账号、邮箱、显示名称和密码。
- 邮箱：配置 SMTP，用于注册、找回密码、验证码邮件。
- 按钮：管理工具箱里的页面、分组和按钮。
- 用户：创建用户、生成邀请码、管理用户状态、下载用户专属工具箱。
- JSON：直接查看或编辑完整配置。
- 对接：下载当前用户专属 EXE，查看工具箱配置接口。

## 2. 登录后台

打开后台地址：

```text
http://你的域名或服务器IP
```

默认总管理员：

```text
用户名：admin
密码：dev-token
```

录屏讲解点：

- 第一次部署后先用默认管理员登录。
- 正式使用前建议修改管理员密码。
- 普通用户可以用邀请码注册，也可以由管理员手动创建。

## 3. 总览：设置工具箱基础信息

进入左侧“总览”。

可以配置：

- 标题：工具箱窗口和登录页显示的名称。
- 副标题 / 官网：工具箱顶部或说明区域显示。
- 版本：当前工具箱版本号。
- 主题：选择工具箱默认主题。
- 工具箱可显示主题数：控制客户端能看到几个主题。
- Logo 文本：左上角或品牌区域显示的短文字。
- 图标链接：后台和客户端显示的图标图片。
- EXE 图标链接：生成客户端 EXE 时使用的图标，建议使用 `.ico` 或 PNG 图床链接。
- 窗口宽度、窗口高度：客户端启动后的默认窗口尺寸。
- 登录页提示：给用户看的登录说明。

设置完成后点击“保存”。

录屏建议：

1. 修改标题和副标题。
2. 选择一个主题。
3. 填写版本号。
4. 点击保存，说明这些内容会同步到新下载的客户端。

## 4. 启动密码

在“总览”的“启动密码”区域：

- 勾选“工具箱启动密码”表示打开客户端时需要输入密码。
- 新密码留空表示不修改当前密码。
- 关闭启动密码后，客户端将不再要求输入启动密码。

录屏讲解点：

- 启动密码是客户端打开时的保护。
- 管理员可以随时修改或关闭。
- 修改后需要让用户重新打开客户端，或重新下载新版客户端测试。

## 5. EXE 属性

在“总览”的“EXE 属性”区域可以设置：

- EXE 文件说明
- EXE 介绍
- EXE 产品名称
- EXE 文件版本
- EXE 公司名称
- EXE 版权

这些信息会写入下载出来的工具箱 EXE 属性里。

录屏建议：

右键下载好的 EXE，打开“属性”，展示文件说明、产品名称和版本号。

## 6. 更新入口

在“总览”的“更新入口”区域可以设置：

- 工具箱更新链接：新版 EXE 的下载地址。
- 更新入口标题：例如“工具箱更新”。
- 更新按钮文字：例如“下载最新版”。

如果更新链接为空，客户端不显示更新入口。

录屏讲解点：

- 当发布新版工具箱时，把新版 EXE 链接填到这里。
- 用户打开客户端后可以看到更新入口并下载新版。

## 7. 按钮管理：位置和分组

进入左侧“按钮”。

先理解三个层级：

```text
位置 -> 分组 -> 按钮
```

位置可以理解为工具箱里的页面或系统工具标签。分组是页面里的分类。按钮是用户真正点击的功能入口。

新增位置：

1. 选择“当前类型”。
2. 填写“新增位置名称”。
3. 点击“新增位置”。

类型说明：

- 普通页面：用于常规网页、下载、命令按钮。
- 系统工具标签：用于客户端里的系统工具区域。

新增分组：

1. 在“选择位置”里选中一个位置。
2. 填写“新增分组”。
3. 点击“新增分组”。

修改名称：

- 选择位置后修改“位置名称”，点击“保存位置”。
- 选择分组后修改“分组名称”，点击“保存分组”。

删除说明：

- 删除位置会删除该位置下所有分组和按钮。
- 删除分组会删除该分组里的按钮，操作前要确认。

## 8. 新增按钮

在“按钮”的“新增按钮”区域操作。

需要填写：

- 位置：按钮放在哪个页面。
- 分组：按钮放在哪个分类。
- 排序：数字越小越靠前。
- 名称：按钮显示名称。
- 图标链接：按钮图标图片地址。
- 按钮介绍：鼠标悬停时显示的说明。
- 动作：按钮点击后执行什么。
- 网址 / 内容：根据动作填写网页地址、下载地址或命令内容。

动作类型：

- 打开网页：填写网址，例如 `https://example.com`。
- 下载文件：填写文件下载直链。
- 运行命令：填写需要执行的命令。
- 内置功能：选择或填写客户端内置功能。
- 安装软件：填写 winget 软件包标识。

录屏建议：

演示新增一个“打开网页”按钮：

```text
名称：官方网站
动作：打开网页
网址 / 内容：https://example.com
```

再演示新增一个“下载文件”按钮：

```text
名称：下载工具包
动作：下载文件
网址 / 内容：https://你的下载地址/文件.exe
```

点击“新增”后，到按钮清单里搜索确认。

## 9. 按钮清单维护

按钮清单支持：

- 搜索按钮名称、分类或网址。
- 按位置筛选。
- 按分组筛选。
- 按动作筛选。
- 修改排序、名称、图标、介绍、动作和目标内容。
- 保存单个按钮。
- 删除单个按钮。

录屏讲解点：

- 排序数字越小越靠前。
- 修改按钮后要点击当前行的“保存”。
- 删除按钮会立即从配置中移除。

## 10. 用户管理

总管理员可以进入左侧“用户”。

新增用户：

1. 填写用户名。
2. 填写邮箱。
3. 填写显示名称。
4. 设置初始密码。
5. 选择角色。
6. 点击“创建用户”。

角色说明：

- 普通用户：只能管理自己的工具箱配置。
- 总管理员：可以管理全部用户、邮箱、邀请码和权限。

用户卡片里可以操作：

- 展开用户详情。
- 修改账号、邮箱、显示名称、角色和密码。
- 开启或关闭 JSON 查看权限。
- 下载该用户专属工具箱。
- 重置工具箱对接地址。
- 删除用户。

批量操作支持：

- 批量启用
- 批量停用
- 批量允许 JSON
- 批量禁止 JSON
- 批量删除

录屏讲解点：

- 每个用户都有独立的工具箱对接地址。
- 重置地址后，旧客户端里的旧地址会失效，需要重新下载客户端。

## 11. 邀请码注册

在“用户”页的“邀请码”区域：

1. 可以填写自定义邀请码，也可以留空自动生成。
2. 设置可用次数。
3. 点击“生成邀请码”。

邀请码支持：

- 启用或停用。
- 删除。
- 批量删除。
- 导出选中邀请码。
- 查看使用次数和使用记录。

录屏讲解点：

- 普通用户注册时必须填写邀请码。
- 邀请码可用次数用完后不能继续注册。

## 12. 邮箱设置

总管理员进入左侧“邮箱”。

填写：

- SMTP 服务器
- 端口
- 邮箱账号
- 发件人
- 授权码 / 密码
- 是否启用 SSL

常见示例：

```text
QQ 邮箱 SMTP：smtp.qq.com
端口：465
SSL：启用
密码：填写邮箱授权码，不是登录密码
```

保存后可以填写“测试收件邮箱”，点击“发送测试邮件”。

录屏讲解点：

- 邮箱配置用于注册、找回密码、发送验证码。
- 授权码一般需要去邮箱后台开启 SMTP 后获取。

## 13. JSON 配置

进入左侧“JSON”。

这里可以查看完整工具箱配置，也可以直接编辑 JSON。

使用建议：

- 普通改按钮、标题、用户时，优先用可视化页面。
- 只有需要批量调整或高级配置时再改 JSON。
- 保存前确认 JSON 格式正确。

录屏讲解点：

- 总管理员可以控制普通用户是否能看到 JSON 页面。
- JSON 修改错误可能导致客户端配置异常，操作前建议备份。

## 14. 对接和下载客户端

进入左侧“对接”。

这里有两个重点：

- 专属工具箱下载：点击“下载当前用户工具箱”。
- 工具箱对接地址：客户端启动时读取配置的接口地址。

下载出来的 EXE 已经写入当前用户的对接地址，用户双击即可打开，不需要手动填写接口。

录屏操作流程：

1. 在后台保存好应用信息和按钮。
2. 进入“对接”。
3. 点击“下载当前用户工具箱”。
4. 打开下载好的 EXE。
5. 展示工具箱标题、主题、按钮和下载功能。

注意：

- 修改后台配置后，客户端重新打开会拉取最新配置。
- 如果修改了 EXE 图标、EXE 属性或客户端代码，需要重新下载新版 EXE。
- 已经发出去的旧 EXE 不会自动改变文件属性和图标。

## 15. 客户端下载记录

客户端支持下载文件，并显示下载任务和下载记录。

功能包括：

- 当前下载进度。
- 暂停、继续、取消。
- 下载记录列表。
- 打开文件。
- 打开目录。
- 删除选中。
- 清空记录。

录屏讲解点：

- 下载中的文件名会单独显示，长文件名可悬停查看。
- 下载记录里可以查看文件名称和保存位置。

## 16. 发布新版客户端

当后台或客户端模板更新后，推荐流程：

1. 部署服务器代码。
2. 重启后台服务。
3. 登录后台。
4. 进入“对接”。
5. 重新下载新版 EXE。
6. 打开新版 EXE 测试。
7. 把新版 EXE 链接填到“总览 -> 更新入口”。

如果服务器路径是 `/www/wwwroot/gjx.vst76.cn`，常用重启命令：

```bash
cd /www/wwwroot/gjx.vst76.cn
systemctl restart toolbox-admin
systemctl status toolbox-admin --no-pager -l
```

## 17. CDN 和安全建议

建议开启 CDN 基础防护，但不要对工具箱客户端接口启用验证码、滑块验证，否则客户端可能无法正常请求。

推荐规则：

```text
域名：gjx.vst76.cn
请求频率：10 秒内总请求 80 次
同 URL 最大请求：20 次
处理方式：请求频率限制或短时拉黑
```

重点保护生成 EXE 接口：

```text
URL 包含：/api/admin/client/download
请求频率：60 秒内总请求 5 次
同 URL 最大请求：3 次
处理方式：短时拉黑 10 分钟
```

服务器安全建议：

- 修改默认管理员密码。
- 只开放必要端口。
- 后台服务建议放在反向代理后面。
- 源站防火墙尽量只允许 CDN 回源 IP 访问。
- 定期备份 `data` 目录。

重要数据目录：

```text
ToolboxAdminApi/data/users.json
ToolboxAdminApi/data/users/<用户ID>/config.json
```

## 18. 推荐录屏顺序

可以按下面顺序录一条完整教程：

1. 打开后台并登录。
2. 介绍左侧菜单。
3. 在总览修改标题、主题、版本并保存。
4. 设置启动密码。
5. 新增位置和分组。
6. 新增一个网页按钮。
7. 新增一个下载按钮。
8. 在按钮清单里修改排序并保存。
9. 创建普通用户。
10. 生成邀请码。
11. 下载当前用户专属工具箱。
12. 打开客户端验证按钮和下载功能。
13. 简单说明 JSON、高级配置和安全建议。

## 19. 录屏口播稿

大家好，这期演示梳理工具箱后台的完整使用流程。

首先我们打开后台地址，使用管理员账号登录。进入后台后，左侧可以看到总览、账号、邮箱、按钮、用户、JSON 和对接几个模块。

总览主要配置工具箱的基础信息，比如标题、副标题、版本、主题、Logo、窗口大小、启动密码、更新入口和 EXE 属性。这里修改后点击保存，客户端重新打开就会读取最新配置。如果改的是 EXE 图标或文件属性，需要重新下载新版客户端。

接下来进入按钮管理。工具箱内容分为位置、分组和按钮三个层级。我们先新建一个位置，再新建分组，然后新增按钮。按钮支持打开网页、下载文件、运行命令、内置功能和安装软件。新增后可以在按钮清单里搜索、筛选、调整排序、修改内容或删除。

用户管理只有总管理员可以看到。这里可以创建普通用户，也可以生成邀请码让用户自己注册。每个用户都有独立的工具箱对接地址和专属 EXE。管理员可以下载任意用户的工具箱，也可以重置用户对接地址。

最后进入对接页面，点击下载当前用户工具箱。下载出来的 EXE 已经写入当前用户的配置接口，双击即可使用。我们打开客户端，可以看到刚才后台配置的标题、主题、按钮和下载功能都已经同步。

这就是梳理工具箱从后台配置到客户端下载使用的完整流程。
