# 梳理工具箱宝塔部署教程

本文档用于把源码部署到宝塔 Linux 面板。推荐部署方式是：Nginx 负责域名和 HTTPS，Python 服务监听本机 `127.0.0.1:5088`。

## 1. 准备环境

服务器建议：

- Linux 系统，推荐 Ubuntu 或 Debian。
- 宝塔面板已安装 Nginx。
- 域名已解析到服务器。
- Python 3 已安装。

生成 Windows 工具箱 EXE 需要 C# 编译器。Ubuntu / Debian 执行：

```bash
apt update
apt install -y python3 mono-devel
```

CentOS / Rocky 可执行：

```bash
yum install -y python3 mono-devel
```

如果系统没有 `mono-devel` 包，需要先配置 Mono 官方源，或先只部署后台，之后再补装 Mono。

## 2. 上传源码

把源码压缩包上传到宝塔网站目录，例如：

```text
/www/wwwroot/gjx.vst76.cn
```

如果上传的是 `toolbox-admin-baota-source.tar.gz`，在宝塔终端执行：

```bash
cd /www/wwwroot
mkdir -p gjx.vst76.cn
tar -xzf /www/wwwroot/toolbox-admin-baota-source.tar.gz -C /www/wwwroot/gjx.vst76.cn --strip-components=1
```

检查核心文件：

```bash
cd /www/wwwroot/gjx.vst76.cn
ls
```

应该能看到：

```text
app.py
assets
client-template
data
deploy
wwwroot
```

## 3. 设置管理员密码

生产环境不要继续使用默认密码 `dev-token`。

如果是第一次部署，设置环境变量里的管理员密码即可。后续也可以登录后台后在“账号”里修改。

推荐密码示例：

```text
大小写字母 + 数字 + 符号，至少 12 位
```

## 4. 创建 systemd 服务

复制服务文件：

```bash
cp /www/wwwroot/gjx.vst76.cn/deploy/baota/toolbox-admin.service /etc/systemd/system/toolbox-admin.service
```

编辑服务文件，把 `TOOLBOX_ADMIN_TOKEN` 改成你的强密码：

```bash
nano /etc/systemd/system/toolbox-admin.service
```

确认这些路径与你的网站目录一致：

```text
WorkingDirectory=/www/wwwroot/gjx.vst76.cn
ExecStart=/usr/bin/python3 /www/wwwroot/gjx.vst76.cn/app.py
Environment=TOOLBOX_HOST=127.0.0.1
Environment=TOOLBOX_PORT=5088
```

启动服务：

```bash
systemctl daemon-reload
systemctl enable toolbox-admin
systemctl restart toolbox-admin
systemctl status toolbox-admin --no-pager -l
```

看到 `active (running)` 表示后台服务已启动。

## 5. 宝塔网站反向代理

在宝塔面板中：

1. 打开“网站”。
2. 找到你的站点，例如 `gjx.vst76.cn`。
3. 打开“设置”。
4. 进入“反向代理”。
5. 添加反向代理：

```text
目标 URL：http://127.0.0.1:5088
发送域名：$host
```

如果你习惯改 Nginx 配置，可以把下面内容放进站点 `server {}` 内：

```nginx
location = /api {
    return 308 /api/;
}

location ^~ /api/ {
    proxy_pass http://127.0.0.1:5088;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header Authorization $http_authorization;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 300s;
    proxy_send_timeout 300s;
    proxy_buffering off;
    expires off;
    add_header Cache-Control "no-store, no-cache, must-revalidate, max-age=0" always;
}

location / {
    proxy_pass http://127.0.0.1:5088;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header Authorization $http_authorization;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

保存后重载 Nginx：

```bash
nginx -t && systemctl reload nginx
```

## 6. 登录后台

浏览器打开：

```text
https://你的域名
```

默认用户名：

```text
admin
```

密码是服务文件里设置的 `TOOLBOX_ADMIN_TOKEN`。

登录后建议立刻进入“账号”修改管理员账号资料和密码。

## 7. 下载专属工具箱 EXE

后台登录后：

1. 进入“总览”，设置标题、主题、图标、版本。
2. 进入“按钮”，添加工具箱按钮。
3. 进入“对接”。
4. 点击“下载当前用户工具箱”。

如果下载 EXE 报错 `服务器未安装 Mono/C# 编译器`，说明服务器缺少 `mono-devel`，执行：

```bash
apt update
apt install -y mono-devel
systemctl restart toolbox-admin
```

## 8. 更新代码

上传新版源码后，在网站目录执行：

```bash
cd /www/wwwroot/gjx.vst76.cn
systemctl restart toolbox-admin
systemctl status toolbox-admin --no-pager -l
```

如果改了客户端模板或 EXE 属性，需要重新进入后台下载新版 EXE。

## 9. 备份数据

最重要的数据目录是：

```text
/www/wwwroot/gjx.vst76.cn/data
```

备份命令：

```bash
cd /www/wwwroot/gjx.vst76.cn
tar -czf /www/backup/toolbox-data-$(date +%Y%m%d%H%M%S).tar.gz data
```

恢复时覆盖 `data` 目录后重启服务：

```bash
systemctl restart toolbox-admin
```

## 10. 安全建议

- 修改默认管理员密码。
- 后台域名开启 HTTPS。
- 服务器防火墙只开放 80、443、SSH 等必要端口。
- Python 服务只监听 `127.0.0.1:5088`，不要直接暴露到公网。
- CDN 不要对 `/api/toolbox/config` 开验证码或滑块，否则客户端可能无法拉取配置。
- 可对 `/api/admin/client/download` 设置较严格的频率限制，防止反复生成 EXE。

推荐 CDN 频率限制：

```text
域名：你的域名
10 秒内总请求 80 次
同 URL 最大请求 20 次
```

生成 EXE 接口建议：

```text
URL 包含 /api/admin/client/download
60 秒内总请求 5 次
同 URL 最大请求 3 次
超过后拉黑 10 分钟
```

## 11. 常用命令

查看服务状态：

```bash
systemctl status toolbox-admin --no-pager -l
```

重启服务：

```bash
systemctl restart toolbox-admin
```

查看日志：

```bash
journalctl -u toolbox-admin -n 100 --no-pager
```

测试本机服务：

```bash
curl -I http://127.0.0.1:5088/
```
